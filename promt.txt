## Architecture Rules

- `services/` — pure business logic, platform-agnostic. No bot imports here.
- `platforms/base.py` — shared abstract interface for all platforms (Bale, Telegram, etc.)
- `platforms/bale/` — Bale-specific bot setup and handlers
- `platforms/telegram/` — reserved for future use, same structure as bale/
- `db/engine.py` — async MySQL connection pool using `aiomysql`, no ORM
- `db/queries.py` — raw SQL query functions
- `db/migrations/` — plain SQL files executed by `migrate.py`
- `utils/helpers.py` — shared utility functions

## Tech Stack

- Python 3.11+
- `python-bale-bot` for Bale platform
- `yt-dlp` for YouTube download/search
- `aiomysql` for async MySQL (no SQLAlchemy ORM)
- `pydantic-settings` + `.env` for config
- Async/await throughout

## Coding Rules

- All code comments must be in English only
- User-facing text (bot messages, button labels, menu options) must be in Persian (Farsi)
- No Persian in code comments, variable names, or docstrings
- Write minimal, clean code — no unnecessary abstractions or boilerplate
- Each file must be self-contained and importable independently
- Follow the structure strictly — never mix service logic into handlers
